# Demo Deliverable Script

Use this when explaining the workflow on camera.

## Script

Okay. You've got good prompts now. With a solid prompt and a character reference, the crew can shoot a great five to fifteen second shot.

So the next question is:

> How do you make a long video with real control over every scene?

The answer is the same technique film directors have used for a hundred years: storyboarding.

Real directors do not show up to set and improvise. They walk in with a shot list. Every shot is mapped before the camera turns on.

We use that same technique for AI video.

Instead of asking the model for one long video, we break the piece into short shots. We write a shot list. We lock the first frame and last frame of every shot before generating video.

The model interpolates between the bookends.

The character holds. The location holds. The story flows.

I built a cooking competition demo ahead of time. The brief was simple: cinematic kitchen, tense music, time pressure, and a pink cup at the chef's station.

Look at the cup. Shot one, shot three, shot five. Same cup. Same color. Same station energy.

That is storyboarding applied to AI video.

Now let me show you how to get organized so you can run this on your own work.

## Visual Cues

- Show the process graphic on the storyboarding explanation.
- Show inconsistent generations when explaining drift.
- Show the storyboard frame grid when saying "first frame and last frame."
- Show a 3-up of the cup across multiple shots.
- End by opening the folder structure.

## What The Demo Must Prove

- The recurring object stays consistent.
- The main character stays consistent.
- The location stays consistent.
- The short clips stitch into one longer sequence.

If the demo does not prove those four things, rerender before recording.
